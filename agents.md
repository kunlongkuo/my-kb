- 以繁體中文回答
- **核心安全守則**：除非用戶明講，不可以刪除原本已建立的任何檔案、不可以做破壞性操作（為執行任務所產生的暫存檔除外）。

## 🟢 開工 / 我來了
- 確認當前目錄是否在 Git 倉庫
- 讀取 Obsidian `每日筆記/` 中的「上次做到哪」與「下一步計畫」
- 執行 `git status`、`git log -n 5`，檢查本地與遠端分支同步狀況
- 給出今日第一步行動建議

## 🔴 收工 / 下班了
- 執行專案敏感檔案安全掃描（防止 API Key 外洩）
- 更新 `README.md`，寫清楚：專案用途、目前功能、啟動方式、部署方式、環境變數、已知問題、下一步
- 更新 `docs/PROGRESS.md`，以日期記錄：今天完成什麼、改了哪些重要檔案、做了哪些決策、目前卡在哪裡、下次接手要先看什麼
- 執行 `git commit` 與 `git push`，且 commit message 必須寫清楚本次改動內容與細節，拒絕簡短無語意的 "update"
- 更新 Obsidian 每日筆記的「已完成工作」與「待辦事項」

## 🔵 初始化專案
- 在根目錄生成 `ANTIGRAVITY.md`、`.gitignore` 與首頁說明檔案
- 執行 `git init`，加入所有檔案並完成 initial commit
- 使用 `gh repo create` 建立遠端 GitHub 倉庫並推上去
- 在 Obsidian 第二大腦中建立對應的專案工作區資料夾

> 以上三個關鍵字（「開工」「收工」「初始化專案」）在對話中說出時，AI 助理會自動觸發相應工作流。

## ⚠️ 主動型 ETF 更新安全規範
- **核心資產保護**：`wiki/金融投資/主動型ETF持股明細.xlsx` 存有歷次更新的完整日期分頁與歷史比對資料。
- **嚴禁行為**：
  1. **禁止**直接複製/覆蓋 `wiki/金融投資/` 下的檔案。
  2. **禁止**在 `skills/active-etf-holdings` 本地執行不帶參數的腳本，這樣會生成一個全新、無歷史分頁的空 XLSX 檔並把歷史覆蓋。
- **標準執行流程**：
  - 務必在**知識庫根目錄**執行，且指定 `--output-dir wiki/金融投資` 參數（現行腳本已改寫為優先自動尋找此路徑）。
  - 這會使腳本讀取現有歷史 Excel，在保留所有舊日期分頁的前提下，無縫追加今日分頁並完成增量歷史比對（新增、刪除、加碼、減碼四大歷史異動分頁自動累加）。
