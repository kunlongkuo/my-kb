# [技術精華] Claude Code 4 小時完整教學：從自動化工作流到知識管理全攻略

這是一份針對 **Claude Code** 的深度指南，重點在於如何利用其本地執行權限，結合自定義技能 (Skills) 與 Obsidian 知識庫，打造全自動化的 AI 工作系統。

🔗 **影片網址**：[https://www.youtube.com/watch?v=fYuohy6rQ9c](https://www.youtube.com/watch?v=fYuohy6rQ9c)

## 💡 核心概述
Claude Code 不僅是一個聊天機器人，它是一個擁有本地電腦讀寫與指令執行權限的 AI Agent。透過建立標準作業程序 (SOP) 形式的「Skills」，使用者可以讓 AI 自動完成複雜的跨平台任務，如資料採集、內容生成及知識分類。

---

## 🛠️ Claude Code 基礎與安裝
*   **與網頁版的差異** [00:23]
    *   **網頁版**：僅能「動口」，無法直接操作檔案或執行指令。
    *   **Claude Code**：擁有最高本地權限，能直接讀取電腦檔案、寫入 Excel、執行 Terminal 指令，實現真正的「動手」。
*   **開發環境配置** [07:23]
    *   推薦使用 **VS Code** 或 Google 的 **IDX (Anti-Gravity)** 作為運行環境。
    *   Claude Code 運行於終端機 (CLI)，能與本地檔案系統無縫連結。
*   **檔案讀取技巧** [10:39]
    *   AI 無法透過「上傳」按鈕拿取檔案，必須提供正確的檔案路徑（Path）。
    *   善用指令讓 AI 自動搜尋與讀取特定路徑下的文檔（如 PDF、圖片或字幕檔）。

## 📂 知識管理：Claude + Obsidian 聯動
*   **Markdown 的重要性** [13:18]
    *   Markdown 格式體積小、易於搜尋，且對 AI 而言消耗的 Token 極低。
    *   Claude 生成的「Artifacts」可直接存為 `.md` 檔案。
*   **建立「有索引」的圖書館** [18:12]
    *   利用 Claude 自動為檔案加上標籤 (Tags) 與分類。
    *   **Obsidain 整合**：將 Claude 的工作目錄設為 Obsidian 的 Vault，實現 AI 自動整理筆記、追蹤進度與繪製關聯圖。
*   **記憶與指令系統** [25:00]
    *   `claude.md`：設定全域或專案級的「行為準則」（如強制回覆繁體中文）。
    *   `memory.md`：記錄工作進度，確保開啟新對話後 AI 仍能接續先前的任務。

## 🚀 核心功能：Skills (AI 自動化工作流)
Skills 是 Claude Code 的靈魂，將複雜任務打包成 SOP。

*   **什麼是 Skill？** [28:04]
    *   它是一套儲存在 `.md` 檔案中的指令集，包含：目的、觸發條件、執行步驟、參考範例。
*   **漸進式載入 (Lazy Loading)** [1:00:43]
    *   AI 只有在收到相關任務時才會讀取特定的 Skill 檔案，有效節省 Token 並避免資訊干擾。
*   **實戰案例：內容生產線** [0:43:00]
    *   **流程**：研究主題 → 數據驗證 → 撰寫多平台文案 → 生成配圖 → 自動上傳至網站。
    *   **成效**：原本需 2 小時的人工任務，AI 自動化只需 10 分鐘。
*   **如何建立自己的 Skill？** [1:06:23]
    1.  **手動測試**：先與 Claude 進行 1-2 次手動對話，修正 AI 的錯誤。
    2.  **提取 SOP**：使用「Skill Creator」指令，讓 AI 根據剛才成功的對話記錄，自動總結出一份 Skill 設定檔。

## 🔍 進階數據蒐集：Firecrawl 爬蟲整合
*   **解決爬蟲封鎖問題** [2:11:35]
    *   傳統 AI 爬網容易遇到 403 錯誤（被封鎖）。
    *   **Firecrawl** 是專為 AI 設計的爬蟲工具，能繞過反爬蟲機制，並將網頁內容轉化為乾淨的 Markdown 格式供 AI 讀取。
*   **應用場景** [2:14:08]
    *   **房地產分析**：從房仲網站抓取最新上架資料。
    *   **競品分析**：自動爬取競爭對手的官網、社交媒體留言，生成深度研究報告。

## 💎 專家建議與決策要點
1.  **逐步微調而非一氣呵成**：不要期待 AI 第一次就能寫出完美的代碼或網站。應先建立框架，再「逐頁」、「逐功能」要求 AI 修正 [2:06:50]。
2.  **善用截圖回饋**：當前端界面 (Front-end) 出錯時，直接截圖傳給 AI，讓它根據視覺反饋修正 CSS 或 Layout [1:47:15]。
3.  **模型選擇**：
    *   **Claude 3.5 Sonnet**：速度快、成本低，適合日常執行任務。
    *   **Claude 3 Opus**：邏輯最強，適合一開始建立複雜架構或除錯 [1:09:40]。
