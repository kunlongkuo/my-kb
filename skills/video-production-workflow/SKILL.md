# 影片製作工作流

先分流，再出架構，最後才進製作。這個 skill 專門把影片需求整理成可執行的工作流。

## 開工

1. 先判斷影片類型。
2. 先讀 [references/video-workflow.md](references/video-workflow.md) 的對應規則。
3. 類型不明時，只問 1 個最小必要問題。

## 產出

### 1. 類型分流

- **活動紀錄影片**：已發生的活動、典禮、賽事、發表會、畢業、婚禮、社區活動。
- **教學影片**：要把概念講清楚，通常是學科、知識、步驟、原理。
- **社群科普影片**：要在 2 到 3 分鐘內抓住注意力，適合 FB、IG、YouTube Shorts、Threads。

### 2. 先出骨架

- 先定義主題、受眾、片長、平台、素材來源。
- 先寫敘事大綱，不要一開始就畫畫面。
- 關鍵節點要停下來，讓使用者確認後再往下。

### 3. 再做結構

- **活動紀錄**：Hook / Setup / Peak / Turn / Echo
- **教學影片**：SOIL 概念定位 -> 脈絡定位 -> 頁面架構表
- **社群科普**：Hook / 比喻 / 機制 / 反轉 / 結語

### 4. 再寫腳本

- 每段或每頁都要有旁白、字卡、畫面重點、動畫意圖。
- 先把內容拆成 beat，再安排素材。
- 避免把講義原文直接搬上畫面。

### 5. 最後才進製作

- 整理素材清單。
- 建立專案結構。
- 準備音訊、圖像、字體與版面。
- 視需要安排渲染與輸出。

## 收工

1. 回頭檢查類型有沒有選對。
2. 回頭檢查結構是不是先於畫面。
3. 回頭檢查字卡、字幕、動畫有沒有各司其職。
4. 如果要更細的規格，就改讀 [references/video-workflow.md](references/video-workflow.md)。

## 核心規則

- 活動紀錄重情緒，不重說教。
- 教學影片重理解，不重花俏。
- 社群科普重 Hook 與節奏，不假設觀眾會開聲音。
- 若使用者已指定類型，以使用者指定為準。
- 若沒有指定，依需求語氣與目的選最接近的類型。
