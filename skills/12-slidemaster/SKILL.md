# 🎬 SlideMaster 投影片轉影片服務

本技能旨在將原本繁瑣的「投影片製作影片」工作流（寫講稿、錄音、對字幕、剪接、輸出）化繁為簡。透過 AI 的三步驟導引，協助使用者快速、一鍵地將靜態投影片（PDF、PPTX 或圖片）轉化為高質感的口語旁白影片。

---

## ⚡ 核心精神：把投影片一路送到「影片」終點線
當用戶完成投影片後，往往面臨四大折磨：
1. **寫講稿**：從零開始，卡很久。
2. **自己錄音**：講錯重錄，超崩潰。
3. **配字幕與剪接**：步驟繁瑣，耗時費力。
4. **輸出影片**：折騰半天，才能拿到結果。

SlideMaster 透過三步驟完美解決：
1. **寫講稿**：AI 依投影片生成流暢口語講稿。
2. **配語音**：AI 合成自然語音，告別手動錄音。
3. **合成影片**：自動對齊投影片、語音與字幕，一鍵合成 MP4。

## 🛠️ 執行流程 (SOP)

當使用者觸發此技能時，請依序執行以下步驟：

### STEP 1：輸入投影片內容並生成「講稿」
1. **收集素材**：引導使用者上傳投影片（PPTX, PDF、或圖片壓縮檔），或是直接貼上每頁投影片的文字大綱/內容。
2. **分析投影片**：
   - 提取各頁投影片的標題、關鍵字與視覺重點。
3. **AI 生成口語講稿**：
   - 避免直接照唸 PPT 的條列式文字，應使用**口語、流暢、像真人說話**的口吻來撰寫每頁的旁白。
   - **格式規範**：
     ```markdown
     ### 📄 第 {N} 頁講稿
     【畫面重點】: {描述畫面中的視覺元素與切入點}
     【口語講稿】: 「{像真人在講話的旁白，字數約 80-150 字，視簡報複雜度調整}」
     ```
4. **確認微調**：請使用者閱讀並修改講稿。使用者確認後，進入 STEP 2。

### STEP 2：配語音 (AI 語音合成)
1. **確認配音風格**：向使用者確認所需的聲音風格（如：專業商務、活潑親切、男聲/女聲、特定語速等）。若有已安裝的語音克隆工具（如 `voxcpm2-voice-cloner`），可主動提議使用。
2. **生成語音**：
   - 將每頁經確認的講稿交給語音合成腳本。
   - 將生成之音訊儲存於臨時目錄中，命名格式為 `audio_{index}.mp3` (例如 `audio_01.mp3`, `audio_02.mp3`...)。
3. **計算時長**：記錄每一段音訊的精確播放長度（秒數），作為後續影片合成的影格停留依據。

### STEP 3：合成影片 (自動對齊與導出)
1. **準備影像**：
   - 若使用者提供的是 PPTX/PDF，需將其導出為每頁的獨立圖檔（如 `slide_01.png`, `slide_02.png`...）。
2. **自動生成字幕 (SRT)**：
   - 依據每頁語音的長度與講稿內容，自動生成對齊的字幕檔（`.srt`），或直接使用 FFmpeg 將字幕燒錄到影片中。
3. **執行影片合成**：
   - 使用 `ffmpeg` 將每頁的圖片與對應的語音音軌合成該頁的單段影片（`.mp4`），確保長度與音訊完全一致。
     - **合成單頁指令範例**：
       ```bash
       ffmpeg -loop 1 -i slide_{index}.png -i audio_{index}.mp3 -c:v libx264 -tune stillimage -c:a aac -b:a 192k -pix_fmt yuv420p -shortest slide_{index}.mp4
       ```
   - 將所有單頁影片合併為最終的完整影片（例如 `output.mp4`）。
     - **合併指令範例**：
       使用 `file_list.txt` 紀錄所有 `slide_{index}.mp4` 的路徑，並執行：
       ```bash
       ffmpeg -f concat -safe 0 -i file_list.txt -c copy final_video.mp4
       ```
4. **燒錄字幕 (選填)**：
   - 若需要將字幕燒入影片，可使用 FFmpeg 濾鏡：
     ```bash
     ffmpeg -i final_video.mp4 -vf "subtitles=sub.srt:force_style='FontSize=20,Alignment=2'" -c:a copy final_output.mp4
     ```
5. **交付影片**：
   - 回報合成結果，提供影片輸出路徑，並引導使用者進行最後確認。

## ⚠️ 安全與設計鐵則
1. **嚴禁空包彈**：如果語音合成或影片合成指令執行失敗，請主動分析 `ffmpeg` 或 TTS 報錯日誌，修正指令後重試。
2. **語音與畫面對齊**：投影片畫面的停留長度必須**完全等於**該頁語音的長度，避免語音播完但畫面已跳頁，或畫面卡住很久卻沒有聲音。
3. **保留歷史備份**：合成前先建立 `workspace/slidemaster_temp/` 目錄，所有生成的中間產物（圖片、語音、SRT、分段影片）均在此處理，避免弄髒使用者的專案目錄。工作完成後，提示使用者是否需要清理暫存檔。
