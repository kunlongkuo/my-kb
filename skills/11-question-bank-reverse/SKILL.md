# 題庫逆向衍生工作流（Question Bank Reverse Engineering）

本技能旨在讀取使用者提供的 Word 題庫檔案，深度分析命題規則與邏輯，再依「概念遷移」原則，憑空生出品質相當的全新試題，最終產出排版整齊的 Word 試卷。

## 先決條件確認

載入此技能後，先確認以下項目：
1. 請使用者確認題庫 `.docx` 檔案是否已放入專案目錄（預設路徑：知識庫根目錄或 `raw/` 資料夾）。
2. 詢問**年段**（國中 / 高中）與**科目**（國文、數學、英文…等）。
3. 詢問**輸出目錄**（預設 `outputDir/`，若不存在則自動建立）。
4. 若使用者沒有提供以上資訊，以合理預設值繼續，並在回報時說明所用假設。

---

## STEP 1：讀取與深度分析題庫

執行以下動作：

1. **讀取 Word 檔**：
   - 使用 Python `python-docx` 套件讀取 `.docx`。
   - 提取所有文字段落，還原題幹、選項（A/B/C/D）與答案（若有標記）。

2. **分析命題結構**，針對每道題目萃取：
   - **命題概念**（考的是哪種能力？如：修辭判斷、句型辨識、詞義推敲、語境理解…）
   - **題型規則**（單選、多選、填充、問答…）
   - **誘答邏輯**（錯誤選項設計原理，如：近義混淆、字形相似、語境錯位…）
   - **難易度評估**（基礎 / 中等 / 進階，依選項複雜度與背景知識門檻判斷）

3. **輸出分析摘要**，以結構化格式回報給使用者：

```
📊 題庫分析摘要
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
共分析 X 題
命題概念分佈：[概念A] x題、[概念B] x題…
難易度分佈：基礎 x 題 / 中等 x 題 / 進階 x 題
主要誘答策略：近義混淆、字形干擾…
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

---

## STEP 2：概念遷移生成全新試題

依分析結果，執行概念遷移出題：

1. **保留命題骨架，更換素材**：
   - 保持與原題**相同命題概念**與**相同難易度**。
   - **不得**複製原題文句，必須另尋全新文句、段落或語境情境。
   - 新素材須**符合指定年段語感**（國中生 / 高中生日常閱讀水準）。

2. **每個命題概念至少生成 3 道新題**，除非使用者另有指定數量。

3. **誘答選項設計守則**：
   - 錯誤選項應具「有邏輯的錯誤」，避免明顯假答案。
   - 保持原誘答策略（如原題用近義混淆，新題也用近義混淆）。

4. **生成後立即回報預覽**，格式如下：

```
📝 第 1 題（命題概念：○○  難易度：中等）
題幹：…
（Ａ）…
（Ｂ）…
（Ｃ）…
（Ｄ）…
✅ 答案：X　解析：…
```

5. **等候使用者確認**，若使用者提出修改（如「第 2 題太難」「換更淺的文句」），立即調整後重新預覽，**不需重新開始全部流程**。

---

## STEP 3：一鍵防呆與排版輸出

當使用者確認題目內容後，執行以下輸出流程：

1. **建立試卷 Word 檔**（使用 `python-docx` 自動排版）：

   試卷結構依序為：
   ```
   [標題行]  ○○年度○○科試卷
   [學生欄位]  班級：＿＿＿  座號：＿＿＿  姓名：＿＿＿＿＿
   [分隔線]
   （ ）1. 題幹文字
          （Ａ）…（Ｂ）…（Ｃ）…（Ｄ）…
   （ ）2. 題幹文字
          …
   ```

2. **排版規範**：
   - 標題：粗體、置中、字型大小 16pt。
   - 學生欄位：標準字型 12pt，左對齊。
   - 題目號碼前空格 `（ ）` 作為作答空間。
   - 每題選項 A/B/C/D 使用全形括號「（Ａ）（Ｂ）（Ｃ）（Ｄ）」。
   - 整份試卷字型統一使用「標楷體」（或系統可用的中文字型）。

3. **輸出路徑**：儲存至使用者指定的 `outputDir/`（預設 `outputDir/新試卷_YYYYMMDD.docx`）。

4. **回報完成訊息**：

```
✅ 試卷已生成完畢！
📁 輸出路徑：outputDir/新試卷_20260624.docx
📊 本次共生成 X 題
💡 小提醒：請開啟檔案再次確認字體與格式，如有版面問題請告知。
```

---

## STEP 4：提供後續調整建議

完成後，主動提示以下可繼續執行的指令：

- **「幫我再加 3 題難題」** → 補充進階題並重新輸出。
- **「幫我做一份答案卷（附解析）」** → 生成含答案與解析的教師版。
- **「調整第 X 題的選項」** → 針對單題修改並更新 Word 檔。
- **「換成填充題格式」** → 將選擇題改為填充題格式重新輸出。

---

## 安全守則

- **嚴禁**在 `SKILL.md` 或任何腳本中寫入 API Key、Token 或個人憑證。
- **嚴禁**覆蓋使用者的原始題庫檔案（`題庫.docx` 等），僅能讀取，輸出一律至 `outputDir/`。
- 輸出前若 `outputDir/` 不存在，自動建立，不報錯。
- 遇到 Word 檔讀取失敗（如加密保護），立即告知使用者並說明解法，不強行繼續。

## 依賴工具

| 工具 | 用途 |
|------|------|
| `python-docx` | 讀取 `.docx` 與生成排版 Word 試卷 |
| Python 3.x | 執行腳本主體 |
| AntiGravity AI | 命題分析、概念遷移、誘答設計 |

若環境中未安裝 `python-docx`，請先執行：
```bash
pip install python-docx
```
