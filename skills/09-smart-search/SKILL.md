# 智慧搜尋（Smart-Search）

本技能旨在為個人知識庫提供高效且精準的多維度全文檢索與內容定位能力，幫助使用者快速找到跨越不同卡片、Wiki 及原始文件的關聯知識。

## 執行流程 (SOP)

當使用者發起智慧搜尋請求時，請執行以下步驟：

### STEP 1：確認搜尋關鍵字與範圍
1. 向使用者確認要搜尋的關鍵字（可支援多個字詞或簡單正則表達式）。
2. 確認搜尋範圍限制（預設為全知識庫，亦可限制在 `wiki/`、`cards/` 或 `raw/` 內）。

### STEP 2：執行搜尋與資料過濾
1. **全文檢索**：使用 `grep_search` 在指定路徑下執行不區分大小寫的搜尋。
2. **優先權排序**：
   - 第一優先：檔名或 Frontmatter 中的 `title` 包含關鍵字。
   - 第二優先：標題（如 `#`、`##`）中包含關鍵字。
   - 第三優先：內文段落包含關鍵字。
3. **過濾非必要項目**：排除自動產生的暫存檔、快取（如 `.venv`、`__pycache__`）或大檔案的二進位資料。

### STEP 3：整合與連結呈現
將搜尋結果整理成 Markdown 格式並呈報。輸出需包含：
- **匹配檔案路徑**（包含 clickable `file://` 連結，例如 `[LLM-Wiki-筆記術.md](file:///i:/Mark/my-kb/wiki/AI%E5%B7%A5%E5%85%B7/LLM-Wiki-%E7%AD%86%E8%A8%98%E8%A1%93.md)`）。
- **符合行號與代碼段摘要 (Line Snippet)**。
- **分類標記**（如 `WIKI`、`CARD` 或 `RAW`）。

### STEP 4：關聯性推薦與探索
根據搜尋到的卡片與 Wiki，分析其雙向連結 (Backlinks)，向使用者推薦 2-3 個可能相關的延伸閱讀卡片或主題頁面。
