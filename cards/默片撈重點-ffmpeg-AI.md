## 核心摘要
本卡片記錄了如何利用 `ffmpeg` 擷取影片關鍵影格，再結合 AI 模型進行 OCR 與結構化整理，將無聲、無字幕、純字卡的快閃短影音快速轉化為可讀筆記的「5 分鐘高效工作流」。

## 關鍵觀點
1. **克服默片閱讀障礙**：無聲字卡影片（如抖音快閃影片）訊息密度高且轉場極快（有時僅 1.5 秒），大腦容易漏看，手動暫停截圖極費時間。
2. **標準化三步驟**：
   - **下載**：將目標影片下載為本機 `.mp4`。
   - **抽影格**：估算字卡段數 $N$，用總長度除以 $N$ 算出時間間隔（如 21 秒影片 / 10 張字卡 $\approx$ 每 2.14 秒一張），用 `ffmpeg` 針對各段中點下指令擷取一張高畫質圖片。
   - **AI 辨識**：將擷取的圖片批次丟給 AI 模型（如 ChatGPT、Claude、Gemini），下達「依序讀出文字並整理成清單」提示詞。
3. **場景變化偵測（進階）**：若影片無字卡而是純動作（如 Vlog、教學示範片），可用 `ffmpeg` 的 `select='gt(scene,0.3)'` 自動偵測畫面變化大於 30% 的瞬間並抓出影格，免去人工對時間點。
4. **知彼解己的教育應用**：呼應史蒂芬·柯維《與成功有約》的第五個習慣。利用這套流程訓練學生拍片與萃取，本質上是「逼自己停下來，把每一個瞬間都看清楚」，培養「先看清楚，再說話」的習慣。

## 下一步行動 / 應用情境
- **教學現場實踐**：預計下學期於八年級課堂實施，引導學生採訪校園人事物並拍攝 1 分鐘默片，隨後用此方法整理並核對訊息。
- **高效知識輸入**：遇到有價值但播放速度極快的無聲音短影音時，停止手動反覆暫停，直接以本流程在 5 分鐘內完成筆記。

## 原始資源參考
- **定點高畫質抽圖**：
  ```bash
  ffmpeg -ss 1.0 -i 影片.mp4 -frames:v 1 -q:v 2 keyframe_01.jpg
  ```
  *(其中 `-ss` 為時間點、`-frames:v 1` 限制張數、`-q:v 2` 維持高品質)*
- **自動偵測場景切換抽圖**：
  ```bash
  ffmpeg -i 影片.mp4 -vf "select='gt(scene,0.3)'" -vsync vfr scene_%03d.jpg
  ```

## 完整流程說明

1. **下載影片**  
   使用 `yt-dlp` 或其他工具將目標影片下載為 `.mp4` 檔案。

2. **估算關鍵字卡數**  
   觀察影片長度，決定大概的字卡張數 `N`（例如 10 張）。

3. **計算抽取時間點**  
   每張字卡的時間間隔 ≈ `影片長度 / N`。  
   以此為基礎，使用 `ffmpeg -ss <時間> -i <影片> -frames:v 1 -q:v 2 <輸出>.jpg` 抽取高畫質影格。

4. **批次抽取**  
   可用簡單的 bash 迴圈或 PowerShell 撰寫腳本，一次產生所有關鍵影格。

5. **AI OCR 與文字整理**  
   - 先將影格上傳至 OCR 模型（例如 Google Vision、Azure Computer Vision）取得文字。  
   - 再將文字送入大型語言模型（如 GPT‑4、Claude、Gemini）以「依序讀出並整理成清單」的提示詞，產出結構化筆記。

6. **後處理與匯出**  
   將整理好的筆記寫入 Markdown、CSV 或 Obsidian 卡片，以便後續檢索與引用。

## 常見問題與除錯

| 問題 | 可能原因 | 解決方案 |
|------|----------|----------|
| 抽取的影格畫質太低 | `-q:v` 數值過高或抽取時間點不在文字區域 | 將 `-q:v` 調整為 2~3，或手動微調 `-ss` 時間 |
| OCR 識別不到文字 | 影格過暗或文字顏色與背景相近 | 加上 `-vf "eq=brightness=0.06:contrast=1.5"` 進行亮度/對比度調整 |
| AI 回答格式錯亂 | 提示詞不夠具體 | 在提示詞中加入「請使用 Markdown 格式」或「每行前加 `-`」等要求 |
| 大量影格導致 API 速率限制 | 同時發送過多請求 | 在腳本中加入排程 (e.g., `time.sleep(1)`) 或使用批次上傳功能 |
