# 一篇文章到80秒說話卡通片完整製作流程

## 核心觀點
透過整合 ChatGPT/Claude、Pollinations.ai (Flux)、MiniMax、HyperFrames 與 Replicate Lyria 2 等 AI 工具，可建構出一套低成本（單支約 NT$18）、高效率（熟練後僅需 5-10 分鐘）的**自動化短影音生成流水線 (Pipeline)**。此流程能將任何文字長文，快速轉換為節奏緊湊、富含暖色 Q 版故事插畫、具備個人克隆人聲及字幕與 BGM 的 80 秒短影音，極具自媒體傳播與教學應用價值。

## 4 大 AI 廚師接力（Pipeline 核心步驟）
1. **剪劇本 (Claude / ChatGPT)**：
   - 將長文重新精簡並拆分為 **23 句字幕**（維持 80 秒長度、平均每句 3.5 秒、不超過 20 個字）。
   - 架構遵循「**Hook (首句強烈吸睛) -> 緣起 -> 卡點 -> 領悟 -> 教訓 -> CTA (結尾訂閱/留言+1)**」的 7 段式黃金故事線。
2. **畫場景 (Pollinations.ai / Flux)**：
   - 生成 **7 張背景插圖**，每張圖對應一個故事段落（平均 3-4 句字幕共用一張背景），避免視覺疲勞且省去過多生成費用。
   - 使用 **Morandi 配色、Flat Illustration、Q-version Cartoon、Paper grain texture** 等關鍵字，以固定主角特徵搭配相同 seed 鎖定角色一致性。
3. **克隆人聲 (MiniMax)**：
   - 錄製 60-90 秒的乾淨個人講話音檔上傳至 MiniMax 克隆出個人專屬語音 ID。
   - **黃金 TTS 設定**：使用 `speech-02-hd` 模型，`speed` 設為 `1.1`（最自然節奏），`emotion` 設為 `calm`（對話感重）。
   - **重要技巧**：繁體中文先以 `opencc` **轉換成簡體中文**送往 TTS 生成（避免中國模型繁體字破音與陸腔），而字幕仍維持繁體呈現。
4. **拼貼渲染與配樂 (HyperFrames + ffmpeg + Replicate)**：
   - **HyperFrames**：HeyGen 開源的網頁渲染工具，直接將編寫好的 HTML/CSS（GSAP 時間軸控制）渲染並錄影輸出為 MP4 影片。
   - **Replicate Lyria 2**：生成商用安全的 110 BPM 輕快鋼琴與吉他原聲背景音樂（Lyria 2 單次生成 30 秒，可用 crossfade 拼接至 90 秒）。
   - **ffmpeg**：以 Python 迴圈自動計算 23 句字幕的毫秒時間戳記，生成 `adelay` 指令將克隆音軌與背景音樂精準混音至 MP4 影片。

## 💡 常見 7 大痛點與避坑指南
* **音頻漏字**：Gemini 轉錄避免單次大檔案（維持 60-90 秒小 chunk 分段）。
* **配音有陸腔**：文字先繁轉簡送 TTS，情緒切換至 `calm`。
* **角色不一致**：Prompt 中加入 character lock 描述 + 固定 seed，或訓練 Flux LoRA。
* **字幕遮擋**：設定字幕在底部 `bottom: 100px`，並加上半透明漸層。
* **首秒黑畫面**：將首張圖的 GSAP fade-in duration 設為 `0` 秒直接亮起。
* **數字符號發音怪**：TTS 輸入字串中將數字轉中文漢字（例：`2026` 轉 `二零二六`、`+1` 轉 `加一`）。
* **結尾硬切突兀**：在 ffmpeg 混音時，於音軌最後 2 秒加上 `afade=t=out` 做淡出處理。

## 🛠️ 兩條實作路線
* **路線 A（無代碼初學者）**：以 ChatGPT 拆劇本、Pollinations 生圖、InVideo/CapCut 內建人聲與素材進行手動拼貼與時間軸對齊。
* **路線 B（自動化進階者）**：配置 Node.js、Python、ffmpeg 及 API 金鑰，使用 Python 腳本（`voice_pipeline_full.py`）一行命令自動完成 TTS 產生、時間軸對齊、GSAP HTML 渲染與音樂合成。
