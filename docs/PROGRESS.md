### 🧭 做了哪些決策

1. **將數據直接輸出為 JS 全域變數**：為了解決瀏覽器本地開啟 HTML 時的 CORS 跨域限制，Python 後台不直接輸出 `.json` 檔案，而是輸出一個 JavaScript 檔案並宣告 `const ETF_DASHBOARD_DATA = ...`。這使得使用者雙擊 `主動型ETF持股分析.html` 即可直接在瀏覽器中使用，不需要額外架設本地 Web 伺服器，大幅提升了 PKM 筆記系統的便攜度。
2. **yfinance 兩階段下載與 OTC 相容處理**：批量下載時台股代號預設帶 `.TW`。下載後檢查有無 NaN 或空數據，若台股 ticker 無資料，在第二波中自動將後綴改為 `.TWO` 重新獲取，確保台股上市、上櫃個股均能順利下載股價走勢。
3. **完全出清訊號手動注入**：當某一期某 ETF 的持股資料消失，後台在該期為該 ETF 注入一筆「完全出清」的記錄（持股量設為 0，張數變動設為負的昨日持股），以便前端能夠正確統計「完全出清」的 ETF 數量並在表格中過濾顯示。
4. **手機端 RWD 與區域網路 QR Code 連線模式**：為了解決手機端無法直接存取 `file://` 本地檔案的問題，提供兩種途徑：其一為在 GitHub Pages 上啟用託管；其二為利用 Python 在本機一鍵啟動內網伺服器，並藉由網頁版 QR Code 讓手機直接掃碼連線，極大簡化了開發與測試時的跨端傳輸門檻。
